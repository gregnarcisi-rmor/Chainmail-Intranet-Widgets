Executive Briefing

Last Updated: June 7, 2026

We have officially transitioned the Chainmail core ecosystem from a sandbox prototype into a 100% production-ready infrastructure. All 13 layers of our "Full-Stack Production Reality" audit are now locked down, verified, and active.

1. Core Architecture Hardening Highlights

  • Live Geocoding & Address Validation: All mock validation constraints have been deleted. The application now queries live, secure Google Address Validation and Geocoding APIs, returning CASS-certified locations.
  • Database & RLS Hardening: Firestore Security Rules are locked. Write operations are restricted at the document level to enforce strict Attribute-Based Access Control (ABAC) based on Google OAuth IDs.
  • Centralized Logging & APM: Application telemetry is instrumented with unique tracing tags (X-Request-ID), routing structured JSON logs directly to centralized performance monitors (Sentry/APM) to track real-time errors.

2. Coworker Beta Testing Phase

With the system hardened, we are launching the internal testing phase to onboarding select partners and team members to test the live platform under controlled conditions.

  1. Access & Authentication: Logins are authenticated securely via Google OAuth, automatically provisioning user profiles in Firestore under /users/{uid}.
  2. Property Registration: Test users register their locations. The geocoder validates the address and returns exact coordinates.
  3. Risk Underwriting: The Underwriting Agent evaluates regional hurricane history and generates a binding quote in under 2 minutes.
  4. KISS/Expert Toggles: Testers toggle between KISS Mode (for clean property setup) and Expert Mode (for viewing historical wind charts and parametric curves).

3. Strategic E-Commerce Selection: Shopify Subdomain

To distribute our white-labeled Sentinel weather sensors and automatically provision nodes on the parametric grid, we are launching a standalone Shopify storefront subdomain: shop.chainmailsolutions.ai.

Why Shopify Subdomain?

PCI-DSS and retail card security are fully outsourced to Shopify, isolating our core underwriting database from transaction liabilities. It natively supports secure checkout webhooks to trigger our node provisioning API routes instantly.

2. Automated Onboarding Sequence

  1. Checkout: Customer buys a Sentinel kit on the Shopify subdomain.
  2. Webhook: Shopify posts an order/paid event containing customer and shipping address data to our API.
  3. Geocoding: API verifies the webhook signature, calls Google Maps API for rooftop coordinates, and saves the property in Firestore as PENDING_ACTIVATION.
  4. Key Generation: System generates a unique activation token, emailing a setup QR code to the customer.
  5. Activation: When plugged in, the device sends telemetry to /api/ingest/ecowitt. Backend matches the token and MAC address, updating the state to VERIFIED_NODE and activating coverage.

2. System Fail-Safes & Error Handling

The Adverse Non-Automation Rule

By base code design, the AI Agents cannot execute autonomous adverse claim actions (denials or cancellations) under NY/FL state laws. When a data mismatch occurs, the agent halts, flags the file, and hands it off to a licensed human Integrity Officer.
